Illuminate Prospecting

In the future integrate huginn, create a data pipeline to do ETL

Our data ingestion stage involves acquiring data from remote sources, pre-processing them and storing them in a staging location. From there, our applications will have their setup code designed to take data from the staging location and make use of them for analysis. Our staging location atm will be AWS. In the future, this staging location could be extended to database service like PostGIS or MonetDB or whatever. Remote sources include any stable location. It's preferable for there to be a stable location, or some direct API access. Pre-processing can mean producing formats that we would like. Like acquiring geojson.

A new generation of Tunnel Mining

Tunnel mining for precious metal

In a tunnel mining operation, assessment of ore quality in the tunnel, guides the drilling direction. Currently this assessment involves geologists manually analysing images taken at the drill. We developed an algorithm to use image data to automatically derive ore quality, and support faster decision making for mining managers and geologists in the context of tunnel mining.

Ore quality assessment while tunneling

Images at various cross sections of the tunnel can help to establish 3D ore distribution map. From which favourable drilling direction can be determined.
